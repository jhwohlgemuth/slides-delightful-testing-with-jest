The Jest API

Much like every other test API
  • matchers leverage expectations on output
  • specs are groupings of matchers for a specific use-case
  • specs can be grouped in suites
  • specs can be unit, regression, or integration(e2e) tests



matchers < specs < suites

Testing Asynchronous Code


    const Promise = require('bluebird');
        

    // return a promise
    it('can wait for promises to resolve', () => {
        expect.assertions(1);
        return Promise.resolve(9001).delay(1000).then(level => {
            expect(level).toBeGreaterThan(9000);
        });
    });
        

    // async/await
    it('can wait for promises to resolve', async () => {
        expect.assertions(1);
        const level = await Promise.resolve(9001).delay(1000);
        expect(level).toBeGreaterThan(9000);
    });
##Snapshot Testing > Comparing serializable values
##details > Before you snapshot deep-fried twinkies... - Snapshots are saved to `__snapshots__` folder - Snapshots are created on the "first pass" - [Snapshots are updated manually](https://facebook.github.io/jest/docs/en/snapshot-testing.html#updating-snapshots) - Snapshots should be commited alongside code changes - [Snapshots should be deterministic](https://facebook.github.io/jest/docs/en/snapshot-testing.html#tests-should-be-deterministic)

Example

Adapted from Monty Python & the Holy Grail 

    const person = {
        weight: 'same as a duck',
        floatsInWater: true,
        madeOfVerySmallRocks: false
    };
        

    // without snapshots
    test('is she a witch', () => {
        expect(person.weight).toEqual('same as duck');
        expect(person.floatsInWater).toBeTruthy();
        expect(person.madeOfVerySmallRocks).toBeFalsy();
    });
            

    // with snapshots
    test('is she a witch', () => {
        expect(person).toMatchSnapshot();
    });

##Visual Regression Testing > with Jest and Puppeteer
## Ingredients - [Jest](https://facebook.github.io/jest/) - *Complete testing solution* - [jest-image-snapshot](https://github.com/americanexpress/jest-image-snapshot) - *Jest plugin* - [puppeteer](https://github.com/GoogleChrome/puppeteer) - *Headless Chrome Node API* - [http-server](https://www.npmjs.com/package/http-server) - *Lightweight http server*

Start static server


    {
        "scripts": {
            "server:start": "nohup http-server -p 1337 &"
        }
    }

Capture Screenshots


    // captureScreenshots.js
    function captureScreenshots(options) {
        return (async () => {
            const browser = await puppeteer.launch();
            const page = await browser.newPage();
            await page.setViewport({width, height});
            await page.goto(/* url */);
            // perform action
            await page.screenshot(/* path */);
            browser.close();
            return 'Capture Complete';
        })();
    }
    captureScreenshots();
            

    {
        "scripts": {
            "capture": "node captureScreenshots.js"
        }
    }

Write tests


  // add image snapshot matcher
  const {toMatchImageSnapshot} = require('jest-image-snapshot');
  expect.extend({toMatchImageSnapshot});
            

  // First run will save snapshots

  // Second run will perform comparisons

  describe('Your application', () => {
      it('can be free of visual regressions', () => {
          return images.forEach(image => {
              expect(image).toMatchImageSnapshot();
          });
      });
  });

Run tests


    {
        "scripts": {
            "server:start": "nohup http-server -p 1337 &",
            "precapture": "npm run server:start",
            "capture": "node captureScreenshots",
            "pretest": "npm run capture",
            "test": "jest"
        }
    }
            

    # from command line
    npm test
